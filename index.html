Military-Grade Protection

ADVANCED SECURITY ARCHITECTURE

Your private keys are protected at every stage — at rest, in transit, and at point of use. DisCryptoBank uses AES-256-GCM authenticated encryption, the same standard trusted by governments and financial institutions worldwide.

🔐 AES-256-GCM ✅ Authenticated Encryption 🧬 Unique IV Per Operation 🛡️ 96-bit Auth Tag
💾

Protected at Rest

All wallet_secret and custodial_secret values in both bot and backend SQLite databases are encrypted with AES-256-GCM. Your raw private key never exists as plaintext in any database.

🔄

Protected in Transit

When secrets are transmitted between the Discord bot and the backend API, they are encrypted before the API call is made. Even if network traffic were intercepted, the payload remains gibberish without the decryption key.

⚡

Protected at Point-of-Use

Private keys are decrypted only at the exact moment a Solana Keypair is needed for transaction signing. After signing, the decrypted key is immediately discarded from memory.

🧮

How AES-256-GCM Works in DisCryptoBank

256-bit Key Strength

AES-256 uses a 256-bit encryption key — yielding 2256 possible combinations. Brute-forcing this would take longer than the age of the universe with current technology.

GCM Authenticated Mode

Galois/Counter Mode (GCM) provides both confidentiality and data integrity. A 96-bit authentication tag ensures any tampering with the ciphertext is immediately detected.

Unique IV Per Encryption

Every encryption operation generates a cryptographically random Initialization Vector (IV). Even encrypting the same key twice produces completely different ciphertext.

How Pot Split Works

Players wager in USD in competitive games — coin flips, trivia, prediction challenges, and more. DCB converts SOL to USD automatically. The pot is split among winners, the host earns a 10% house cut, and payouts are routed through DisCryptoBank's encrypted transaction system. Since users pay into the pot to join, payouts are always guaranteed.

01

Host Creates a Game

Choose game type (coin flip, trivia, photo vote), set the wager amount per player, and max participants.

02

Players Join & Wager

Each participant stakes their wager from their encrypted user wallet. Funds are held securely in the pot during the game.

03

Game Plays Out

The game runs within Discord — votes are cast, answers submitted, or predictions made. All actions are logged.

04

Host Earns 10% — Winners Get the Rest

The host's 10% house cut is automatically deducted, then the remaining pot is split among winners. Encrypted wallets are decrypted only for the payout transaction, then wiped.

FREQUENTLY ASKED QUESTIONS

Is the server treasury changeable?

Treasury setup is designed as a one-time configuration for safety. This prevents accidental misroutes and ensures financial integrity.

Do users need a wallet on every server?

No. /user-wallet sets a personal receiving wallet that works across all servers running DisCryptoBank.

How do DCB Events determine winners?

The server owner privately nominates a winning image. Any participant who votes for that image is declared a winner and receives an automatic payout.

What is the DCB Event Manager?

A monthly subscription web dashboard that gives server owners a full UI for staff management, payroll scheduling, and transparent ledger audits.

Does DCB only work with Discord?

Discord is the primary platform today, with the new DCB Pot Split Android app extending functionality to mobile. DisCryptoBank's payroll engine is built on APIs and Solana blockchain — designed to expand to Telegram, Slack, web apps, and more.

How does the 10% host house cut work?

When you host a Pot Split game via the Android app, DCB automatically deducts 10% of the total pot as your house cut before distributing winnings. Since all participants pay into the pot upfront, payouts are always fully funded — no risk, no manual collection.

Is the DCB Pot Split Android app secure?

Absolutely. The app uses full end-to-end encryption (AES-256-GCM) for all wallet and private key operations. Keys are never stored in plaintext and are only decrypted for milliseconds during transaction signing, then immediately wiped from memory.

How do buyers see progress?

DisCryptoBank surfaces generated vs assigned vs pending completion — and approved proof screenshots can be displayed on IllySocial as live testimonials.
